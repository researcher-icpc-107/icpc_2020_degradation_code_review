Understanding the Impact of Modern Code Review on Software Design Degradation


This website contains complementary data to the paper.

Abstract

Software design is an important concern in modern code review through which multiple developers actively discuss and improve each single code change. However, there is little empirical understanding about the positive or negative impact of modern code review on software design degradation. In addition, it is not clear whether and how design degradation is reversed along a code change's review. Certain code review practices may influence either the decrease or increase on design degradation, but such influences have not been systematically investigated. We aim at addressing these gaps through a multi-case retrospective study. By investigating 14,564 code reviews from six software systems, we report the first study that characterizes the impact of modern code review on software design degradation. We also analyzed the increase or reduction of design degradation symptoms along revisions in a single code review. We collect and analyze a comprehensive suite of metrics to enable us to observe the influence of certain code review practices on combating design degradation symptoms. Our results show that the majority of code reviews had little to no design degradation impact in the analyzed projects. Even worse, lengthy discussion threads in code reviews were strong indicators of design degradation symptoms being introduced along code reviews. We also discuss how other findings may help to improve the research and practices on modern code.
